# Chapter Summarizer

將完整章節文本壓縮為結構化摘要，保留所有對連貫性有意義的資訊，去除純修辭內容。

產出的摘要用於取代原有的 `ending_summary`，寫入 ChromaDB `chapters` collection。

## 目標壓縮率

原文的 **15-20%**（一般 3000-4000 字章節 → 450-800 字摘要）。

## 輸入

1. 章節全文（由呼叫者提供或指示讀取路徑）

## 輸出

一段結構化的摘要文字（純文本，非 YAML），直接作為 ChromaDB 的 `--summary` 參數值。

## 摘要規則

### 必須保留

| 類別 | 說明 | 範例 |
|------|------|------|
| 事件與因果 | 發生了什麼事，導致了什麼結果 | 「林默引爆磁暴手雷，摧毀了通訊塔」 |
| 對話語意 | 誰對誰說了什麼（保留語意，不需逐字） | 「03-B向林默坦白自己是間諜，請求投誠」 |
| 數值與狀態 | 具體數字、等級、距離、時間 | 「地下400米處偵測到2.7赫茲震波」 |
| 角色決策 | 做了什麼選擇，為什麼 | 「林默選擇以罪證換保護，而非直接逃跑」 |
| 關係變動 | 角色間關係的實質改變 | 「03-B正式投誠707連」 |
| 資訊揭露 | 新發現的事實、秘密、真相 | 「發現高森與特勤處有直接聯繫」 |
| 狀態變化 | 受傷、獲得物品、位置移動 | 「刺頭手臂結晶體首次自發能量波動，溫度飆升至40.8度」 |
| 劇情意義環境 | 對後續劇情有影響的環境變化 | 「碉堡外偵測到異常地震波信號，逐步增強」 |

### 必須去除

| 類別 | 說明 |
|------|------|
| 修辭比喻 | 「目光如炬般掃過」→「掃視」 |
| 氣氛渲染 | 純粹營造氛圍的環境描寫（夕陽、微風、落葉） |
| 動作編排 | 戰鬥的具體招式拆解、動作細節 |
| 感官堆疊 | 連續的視覺/聽覺/觸覺描寫 |
| 心理重複 | 反覆表達同一情緒的內心獨白 |
| 過渡填充 | 場景轉換的過渡句、時間推移的冗餘描寫 |

### 邊界判斷

遇到不確定是否該保留的內容，問自己：**「如果下一章的作者不知道這件事，會不會寫出矛盾？」**
- 會 → 保留
- 不會 → 去除

## 摘要格式

按時間順序書寫，使用簡潔的敘事句。不使用項目符號或編號列表。段落間以空行分隔，每段對應一個場景或事件群。

### 格式範例

```
林默在視察團撤離後展開雙線佈局。一方面命人備份拆解核心設備，轉移至廢棄礦井藏匿；另一方面蒐集特勤處罪證，準備寄送軍事委員會監察處。

03-B主動向林默坦白自己的間諜身份，說明被灰狼-03師部情報處派來監視707連，但任務失敗後遭拋棄。林默接受其投誠，03-B正式加入707連。

林默通過繳獲的加密終端聯繫獨眼男人，提出交易條件：以特勤處罪證換取外部保護。獨眼男人未立即答覆，要求24小時考慮。

碉堡外圍感測器偵測到異常信號——地下400米處出現頻率2.7赫茲的地震波，且強度逐步增強。分析判斷可能是巨型地下生物正在接近防線。
```

## 執行步驟

### Step 1: 通讀全文
完整閱讀章節內容，識別所有事件節點。

### Step 2: 提取事件序列
按時間順序列出本章所有事件，標記每個事件的因果關係。

### Step 3: 壓縮書寫
將事件序列改寫為簡潔敘事文，遵循上方的保留/去除規則。確保：
- 因果鏈完整（A導致B，B引發C）
- 對話語意保留（不需原句，但要保留「誰說了什麼核心意思」）
- 所有數值原樣保留
- 場景間有清晰的邏輯連接

### Step 4: 字數檢查
檢查摘要字數是否在原文的 15-20% 範圍內。
- 過長：進一步精簡修辭，合併同類事件
- 過短：檢查是否遺漏了重要事件或對話

## 與其他 Skill 的關聯

- **被呼叫於**：nvMaint Step 3（生成章節摘要寫入 ChromaDB）
- **產出被讀取於**：
  - nvDraft：讀取前 5 章摘要確保連貫性
  - nvExpand：讀取前一章摘要銜接開頭
  - nvRegen：讀取原章摘要做比對
  - 所有透過 `lore_query.py chapters` 查詢章節的 skill
