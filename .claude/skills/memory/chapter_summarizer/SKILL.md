# Chapter Summarizer

將章節文本壓縮為結構化摘要（原文 **15-20%**），作為 ChromaDB `--summary` 參數值。

## 保留 vs 去除

| 保留（影響連貫性） | 去除（純修辭） |
|---|---|
| 事件與因果鏈 | 修辭比喻、氣氛渲染 |
| 對話語意（誰對誰說了什麼核心意思） | 動作編排細節 |
| 具體數值（距離/時間/等級） | 感官堆疊、心理重複 |
| 角色決策及原因 | 過渡填充句 |
| 關係變動、資訊揭露 | |
| 狀態變化（受傷/獲物/位移） | |
| 對後續有影響的環境變化 | |

> 邊界判斷：「下一章作者不知道這件事，會不會寫出矛盾？」會 → 保留，不會 → 去除。

## 格式

按時間順序，簡潔敘事句，段落間空行分隔（每段對應一個場景/事件群）。不用項目符號或編號。

## 執行步驟

1. **提取事件序列**：按時間順序列出所有事件，標記因果關係
2. **壓縮書寫**：改寫為簡潔敘事文，遵循保留/去除規則。確保因果鏈完整、對話語意保留、數值原樣保留
3. **字數檢查**：15-20% 範圍。過長→精簡修辭/合併同類；過短→檢查遺漏
4. **聲音快照**（附在摘要末尾，1-2 行）：
   - 本章旁白主要語氣（如：吐槽/冷靜/戲謔）
   - 對話風格（如：嘴炮密集/沉默為主/正式交涉）
   - 特殊語氣標記（如：某角色本章出現新口頭禪）
