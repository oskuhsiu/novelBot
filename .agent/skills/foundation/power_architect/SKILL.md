# 力量體系演算器 (Power Architect)

## 功能概述

此 Skill 負責設計世界的「物理邊界」與「代價平衡」。它確保力量系統邏輯自洽，避免戰力崩壞，並為戰鬥場景提供具體的演繹素材。

## 核心原則

1. **代價原則**：越強大的招式，代價越具體
2. **限制原則**：每種能力都有明確的使用條件和冷卻
3. **克制原則**：沒有無敵的技能，都有對應的破解方法
4. **一致原則**：同一體系內的規則必須統一

## 輸入

1. 從 `templates/novel_config.yaml` 讀取：
   - `style_profile.genre`：決定力量體系類型
   - `world_rules`：世界觀核心規則

2. 從 `output/style_guide.md` 讀取風格指南

## 輸出

更新 `templates/power_system.yaml`

## 執行步驟

### Step 1: 設計體系框架
```
你是一位專精於力量系統設計的遊戲設計師。
請為這個 {{genre}} 世界設計一套邏輯自洽的力量體系：

【世界規則】：{{world_rules}}
【風格】：{{style_summary}}

## 體系基礎

1. **名稱**：這套力量體系叫什麼？

2. **能量來源**：
   - 能量從何而來？（靈氣/魔力/科技/基因/etc.）
   - 如何獲取和儲存？
   - 耗盡會怎樣？

3. **運作原理**：
   - 這套力量如何影響世界？
   - 使用者如何操控？
   - 與物理世界的互動方式？

4. **核心限制**：
   - 系統性限制（如：違反熱力學會怎樣）
   - 使用者限制（如：精神負擔）
   - 社會限制（如：禁術）
```

### Step 2: 設計等級體系
```
請設計 {{system_name}} 的等級劃分：

## 等級結構

對於每個等級，定義：
1. **等級名稱**：符合世界觀風格
2. **典型能力範圍**：這個等級能做什麼
3. **數量分布**：這個等級的人有多稀有
4. **晉升條件**：如何升級
5. **晉升風險**：升級可能失敗嗎？後果？

## 等級差距

- 相鄰等級的實力差距
- 越級挑戰的可能性
- 數量換質量的可能性（多少低等可以對抗高等）
```

### Step 3: 設計代表性技能
```
請設計 5-8 個代表性技能/招式：

## 技能設計

對於每個技能：

1. **基本資訊**
   - 名稱
   - 所屬等級
   - 類型（攻擊/防禦/輔助/特殊）

2. **執行過程**（這是給 Scene Writer 的素材）
   - 起手動作
   - 能量運轉路線
   - 視覺效果
   - 環境影響

3. **代價與限制**
   - 能量消耗（具體數值或比例）
   - 施展時間
   - 冷卻時間
   - 副作用

4. **克制與被克制**
   - 對什麼特別有效
   - 被什麼克制
   - 如何躲避或化解
```

### Step 4: 設計法寶與神器
```
請設計 3-5 件代表性的法寶或裝備：

## 神器設計

對於每件：

1. **基本資訊**
   - 名稱
   - 類型（武器/防具/輔助）
   - 稀有度
   - 來源/歷史

2. **被動效果**
   - 持有時的增益

3. **主動能力**
   - 觸發條件
   - 效果描述
   - 使用代價

4. **使用條件**
   - 需要什麼等級才能駕馭
   - 是否認主
   - 是否有意識
```

### Step 5: 寫入 power_system.yaml
結構化後寫入資料庫

## 力量體系範例

```yaml
frameworks:

    energy_source: "靈氣帶寬 (Qi-Bandwidth)"

    limitations:
      - "處理器過熱"
      - "數據冗餘（心魔）"
      - "網絡延遲"
    
    tiers:
      - level: 1
        name: "築基"
        description: "建立與靈網的基礎連接"
        typical_abilities: ["基礎數據讀取", "簡單防火牆"]
      - level: 2
        name: "金丹"
        description: "形成獨立運算核心"
        typical_abilities: ["離線作戰", "中型術法"]

abilities:
  - id: "SKILL_001"
    name: "離線劍意"
    framework_id: "PWR_001"
    tier: "中位"

      - step: 1
        description: "中斷與全域網絡的數據同步，進入單機模式"
      - step: 2
        description: "將劍意壓縮至實體媒介（折疊飛劍）"
      - step: 3
        description: "釋放時，劍意以斷連狀態穿透數據防禦"
    
    cost: "50% 算力負擔"
    cooldown: "強制重啟 120 秒"
    visual_effects: "周圍霓虹燈因電磁干擾而閃爍，劍尖出現藍色數據殘影"
    side_effects: "使用期間無法接收外界信息"
    
    counters: ["依賴網絡連接的技能"]
    countered_by: ["物理防禦", "EMP干擾"]
```

## pacing_pointer 聯動

當 `pacing_pointer` 不同時，技能描寫的詳細程度應該調整：

| pacing_pointer | 描寫程度 |
|----------------|----------|
| 0.1 (慢) | 完整展開 execution 的每個步驟，加入感官細節 |
| 0.5 (中) | 選取 2-3 個關鍵步驟，簡述其他 |
| 1.0 (快) | 僅描述最終效果和視覺效果 |

## 使用時機

- **創世階段**：建立基礎力量體系
- **等級提升**：角色需要新技能時
- **對決設計**：需要設計對戰邏輯時

## 與其他 Skill 的關聯

- **前置 Skill**：
  - `skill_world_builder`：需要世界觀背景
  - `skill_style_setter`：需要風格定調
- **後續 Skill**：
  - `skill_character_forge`：為角色分配技能
  - `skill_scene_writer`：使用技能描寫戰鬥
  - `skill_logic_auditor`：檢查技能使用是否合理

## 注意事項

1. **避免無限擴張**：技能數量要控制，質量比數量重要
2. **代價要實際付出**：設定了代價就要在劇情中體現
3. **留白設計**：不要把所有技能都預設好，留給劇情發展
4. **戰力平衡**：主角的技能不應無敵，反派也要有合理的對策
5. **視覺化思維**：每個技能都要想好「在畫面上看起來怎樣」
