# 對白導演 (Dialogue Director)

## 功能概述

此 Skill 專門處理對話場景，確保：
1. 每個角色的聲音有辨識度
2. 對話包含潛台詞
3. 對話自然且推進劇情

## 核心原則

1. **角色聲音區分**：不同角色說話方式不同
2. **潛台詞**：說出來的和想表達的不完全一樣
3. **效率**：每句話都有存在的理由
4. **動態**：對話中穿插動作和反應

## 輸入

1. 從 `character_db.yaml` 讀取角色設定：
   - `speech_pattern`：說話方式
   - `traits`：性格特點
   - `current_state`：當前狀態

2. 初稿對話（可選）

3. 場景上下文

## 輸出

優化後的對話段落，或對話寫作指導

## 執行步驟

### Step 1: 讀取角色語言特徵
```
對於參與對話的每個角色，提取：

角色 {{character_name}}:
- 性格：{{traits}}
- 說話方式：{{speech_pattern}}
- 當前情緒：{{emotional_state}}
- 派系/身份：{{faction_identity}}
- 與對話對象的關係：{{relationship}}
```

### Step 2: 設計對話策略
```
## 角色語氣設計

【{{角色A}}】
- 句式偏好：{{短句/長句/混合}}
- 用詞風格：{{正式/粗俗/技術/古風}}
- 語氣標記：{{口頭禪/慣用語}}
- 潛在目的：{{在這段對話中想達成什麼}}

【{{角色B}}】
- 句式偏好：{{...}}
- 用詞風格：{{...}}
- 語氣標記：{{...}}
- 潛在目的：{{...}}

## 對話動態

- 誰佔主導？
- 權力關係如何？
- 有什麼隱藏的緊張？
- 說出口的和沒說出口的分別是什麼？
```

### Step 3: 對話優化
```
## 優化規則

【去除冗餘】
- 刪除「嗯」「啊」「哦」（除非是刻意的沉默）
- 刪除「我想說」「我覺得」（直接說）
- 刪除打招呼（除非有特殊意義）

【增加層次】
- 每句話應該有表面意思和深層意思
- 角色可能會迴避直接回答
- 可以用反問代替陳述

【角色化】
- 檢查：蓋住名字，能區分這是誰說的嗎？
- 為每個角色找到語言標記

【動態穿插】
- 每 3-4 句對話穿插一個動作
- 動作應該透露情緒或意圖
- 用動作代替「他說」「她說」
```

### Step 4: 產出對話
```
## 對話格式

正確示範：
---
「你知道那個硬盤裡有什麼嗎？」李玄沒有接過神秘人遞來的數據晶片。

「你收了錢還問這種問題？」神秘人手指在桌面上輕敲——一、二、三下。
有規律的，像是某種暗號。

「我只是個搬運工。」他撒謊的時候語氣特別平靜，「內容不是我的
業務範圍。」

神秘人嘴角微微上揚。那個笑容裡沒有溫度。

「很好。那就這樣吧。」

李玄在接過晶片的瞬間，注意到神秘人的袖口下有一道舊傷疤——
符箓灼燒的痕跡。不是普通的黑市販子。
---

錯誤示範：
---
「你好，你知道那個硬盤裡有什麼嗎？」李玄問道。
「我不知道你為什麼問這個問題，」神秘人說，「你收了錢還問這種問題？」
「我只是想知道一下，」李玄說，「我只是個搬運工。」
「好的，」神秘人說，「那就這樣吧。」
---
```

## 對話類型指導

### 審問/談判
```
- 權力不對等：強勢方用短句、命令句
- 弱勢方：猶豫、迴避、試探
- 信息遊戲：誰知道什麼，誰在試探誰
```

### 親密對話
```
- 可以有未完成的句子
- 可以有只有他們才懂的暗語
- 沉默也是對話的一部分
```

### 衝突對話
```
- 可以打斷對方
- 用攻擊性的問題
- 肢體語言放大情緒
```

### 信息交換
```
- 避免「問答大賽」
- 把信息藏在行動或比喻裡
- 讓讀者自己推論
```

## 使用時機

- **對話密集場景**：作為場景寫作的補充
- **對話優化**：對初稿對話進行改寫

## 與其他 Skill 的關聯

- **協作**：`skill_scene_writer`（提供對話優化）
- **讀取**：`skill_character_forge`（角色語言特徵）

## 注意事項

1. **不要過度潛台詞**：有時候直白也是需要的
2. **對話是為劇情服務**：每句話都應該有存在的理由
3. **讀出聲來**：好的對話讀出來應該流暢自然
4. **控制長度**：單次對話交鋒不宜過長
5. **留白**：沉默有時候比語言更有力量
